# 目标检测评估指标

## 基础概念

### TP、FP、FN
- **TP（True Positive）**：正确检测到目标，且与真实目标匹配
- **FP（False Positive）**：错误检测，将背景或其他类别误判为目标
- **FN（False Negative）**：漏检，未检测到真实存在的目标

### IoU 阈值
- 通常设置为 0.5，即预测框与真实框的 IoU ≥ 0.5 时认为匹配成功
- 严格评估时会使用多个阈值（0.5~0.95）

## Precision（精确率）

精确率衡量检测结果的准确性：

Precision = TP / (TP + FP)

- 精确率高：检测出的目标大部分是正确的
- 精确率低：存在较多误检

## Recall（召回率）

召回率衡量检测的完整性：

Recall = TP / (TP + FN)

- 召回率高：大部分真实目标都被检测出来
- 召回率低：存在较多漏检

## Precision-Recall 曲线

PR 曲线展示不同置信度阈值下精确率和召回率的关系：
- 曲线越靠近右上角，模型性能越好
- 曲线下面积（AP）是评估模型的核心指标

## AP（Average Precision）

AP 是 PR 曲线下的面积，取值范围 0~1：
- AP = 1：完美检测，无漏检无错检
- AP = 0：完全无法检测

## mAP（mean Average Precision）

mAP 是所有类别 AP 的平均值：
- **mAP50**：IoU 阈值为 0.5 时的 mAP
- **mAP50-95**：IoU 阈值从 0.5 到 0.95（步长 0.05）的平均 mAP

## F1 Score

F1 分数是精确率和召回率的调和平均：

F1 = 2 × Precision × Recall / (Precision + Recall)

- F1 高：精确率和召回率都高
- F1 低：精确率和召回率至少有一个较低

## 实际应用中的权衡

在实际应用中，精确率和召回率需要权衡：
- **质检场景**：希望精确率高，避免误检导致的误判
- **安防场景**：希望召回率高，避免漏检导致的安全隐患
- **平衡场景**：使用 F1 分数综合评估
